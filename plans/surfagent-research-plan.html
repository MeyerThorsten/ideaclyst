IdeaClyst × surfagent

Web Research & Idea Discovery — grounding the council in live web data and turning IdeaClyst into a way to find ideas, not just evaluate them.

Vendor surfagent CDP 4 features Mock-first, best-effort live +1 dep: chrome-remote-interface

Context

IdeaClyst currently turns a given idea into a founder packet via a 5-step Claude+Codex council, but it has no contact with the real world — strategy and validation rest on the models' priors. surfagent (sibling project at /Users/thorstenmeyer/Dev/surfagent) is a pure-CDP browser-automation toolkit that headlessly opens pages and returns structured "recon" data. By vendoring surfagent's read-only CDP modules into IdeaClyst we can ground the council in live web data and add a way to discover ideas.

Goal (per user): build all four capabilities in one pass, vendoring the code:

  1. Market Research (Step 0) — ground the council in real web data.
  2. Idea Discovery mode — scout sources to propose & rank candidate ideas.
  3. Validation evidence — back the Validation Tests section with real signals + links.
  4. Competitor teardown — recon competitor landing pages to sharpen differentiation.

Hard constraints (do not regress): the only mode seam stays IDEACLYST_AGENT_MODE (mock default / cli) — research follows it (cli→live Chrome, mock→offline) with an optional IDEACLYST_RESEARCH_MODE override. Research is best-effort and never throws into a run (missing/blocked Chrome → labeled note, council still completes). Default mock experience stays fully offline; disk stays the source of truth.

Architecture

Vendor surfagent's pure-CDP, read-only modules into src/lib/research/, wrap them in a lazily-launched, reference-counted headless Chrome lifecycle manager, and expose one mode-aware seam (src/lib/research/index.ts) mirroring src/lib/agents/index.ts. Live research synthesizes raw recon snippets into Markdown via the existing runAgent("claude", …) call; mock research returns deterministic idea-aware Markdown. Features A/C/D extend the existing council; Feature B is a parallel discovery store + orchestrator that reuses the research seam and createRun/startRun to promote a chosen candidate into a normal run.

Risks & mitigations

  • Anti-bot / captcha / headless detection → DuckDuckGo HTML default; never solve captchas; degrade on captchas/empty results; Bing fallback.
  • Chrome missing / wrong path → caught → degrade to mock-with-note; run never fails; IDEACLYST_CHROME_BIN + auto-detect.
  • Research stalls a run → per-recon timeout + overall Promise.race budget + caps.
  • Chrome/tab leaks → single refcounted singleton, keepTab:false, idle reaper, exit cleanup, dedicated profile.
  • Prompt injection from scraped pages → delimited "untrusted data" block in synthesis prompts.
  • Login walls (Reddit/Product Hunt) → each source best-effort, skipped-with-note.

Verification (end-to-end)

  1. npm run typecheck && npm run lint — clean.
  2. Mock default (offline): create a run (optionally with competitor URLs) → Research tab populates first, council completes, Validation Tests cite (mock) evidence.
  3. Live path: IDEACLYST_AGENT_MODE=cli (Chrome present) → real DuckDuckGo-sourced links in RESEARCH_FINDINGS.md; a broken IDEACLYST_CHROME_BIN still completes with a degraded note.
  4. Discovery: /discover → candidate cards (deterministic in mock); "Promote to council" lands on a real run page.
  5. No orphan Chrome after idle (pgrep -f remote-debugging-port).
